Detailed Description

Generic CatalogStore implementation on top of sqlpp23.

The query bodies are identical across connectors: sqlpp23 serializes the same typed statements into each SQL dialect automatically. Only two things vary per connector, both supplied by the Traits policy:

  • how driver errors are recognized as unique-constraint violations (Traits::IsUniqueViolation).
  • how driver errors are recognized as duplicate-column migrations (Traits::IsDuplicateColumn).

This is an internal header that pulls in sqlpp23; it is only included by the per-connector factory translation units (catalog_store_*.cc).
